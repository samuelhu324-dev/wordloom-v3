# 一、亮 / 暗色主题 UI 评分标准 v1（对外版）

> 面向普通用户入口、首页、主要工作界面，用来判断“好看、好用、好卖、诱人”的程度。

## 0. 统一打分区间

所有维度分数 `Lᵢ ∈ {0,1,2,3,4,5}`：

- **0 分**：几乎完全缺失 / 直接劝退用户
- **1 分**：非常弱，只在局部勉强出现
- **2 分**：明显不足，有大量违和感或阻碍
- **3 分**：中等，可用但不吸引，有清晰改进空间
- **4 分**：良好，整体体验顺滑，细节有少量可优化
- **5 分**：优秀，情绪 + 功能都打中位，容易“让人想多用一会儿”

---

## 1. 维度与权重

### L₁：第一印象与诱人程度（w₁ = 1.3）

- 打开 3 秒内，用户能否感到：“这里值得停留一下”？
- 主视觉是否有明确情绪（专业、亲切、冷静…），而不是杂乱模板感？
- 是否有清晰的“我接下来可以做什么”的视觉引导（hero CTA、主入口）？

---

### L₂：简单易学与认知负担（w₂ = 1.5）

- 首次使用是否需要阅读大量说明才能上手？
- 操作控件是否“看起来就知道怎么用”？
- 是否避免了不必要的术语、复杂设置和多余步骤？

---

### L₃：任务流与操作效率（w₃ = 1.3）

- 常见任务（搜索、创建、编辑、切换视图）路径是否短而直？
- 是否有明显减少重复操作的设计（默认值、上次状态记忆、快捷入口）？
- 亮 / 暗色主题切换后，关键任务流是否保持一致节奏，不被干扰？

---

### L₄：视觉层级与品牌气质（w₄ = 1.2）

- 标题、正文、按钮、说明文是否有清晰层级（大小、粗细、颜色、间距）？
- 颜色、字体、icon 风格是否统一，能否被描述成一种清晰的品牌气质？
- 亮 / 暗色两套主题是否感觉是同一家产品，而不是两个随机皮肤？

---

### L₅：可发现性与转化驱动（w₅ = 1.2）

- 新用户是否容易发现：创建、导入、试用核心功能的入口？
- 付费 / 升级 / 关键行为按钮是否醒目但不过分吓人？
- 是否有适度的“下一步推荐”（空状态引导、提示卡），推动用户往前走？

---

### L₆：舒适度与长期使用疲劳（w₆ = 0.9）

- 亮色主题：背景是否过白刺眼、对比过强？
- 暗色主题：是否避免纯黑 + 纯白极端对比，文本是否仍然清晰？
- 长时间停留（>1 小时）是否容易产生“视觉噪音疲劳”（太花、太密、太闪）？

---

### L₇：主题适配与环境弹性（w₇ = 0.6）

- 在不同屏幕尺寸下（笔记本 / 外接屏宽度变化）布局是否稳定？
- 亮 / 暗主题在同一页面内切换时，是否保证：
  - 重点不位移、操作位置不变、只换情绪不换结构？
- 是否考虑系统级暗色模式 / 环境光，避免完全相反的观感冲击？
