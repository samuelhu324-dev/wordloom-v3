一、先理顺信息结构（建议你就按这个改）

1. 顶部 tab：只做「页面级」模式

保持现在：

概览（Overview）

块编辑（Blocks）

时间线（Timeline）

这三个是 Book 的三个大视图，就让它们当唯一的一级 tab，不再出现第二条长条 nav。

2. 概览里的几个内容区：做成「卡片 + 局部切换」

你下方那条：查看评分构成 / 结构任务 / 时间线 / TODO
其实都是跟「成熟度 & 事件」相关的视图，可以收编到一个卡片里：

卡片标题：评估 · Maturity insights（随便起，好记就行）

卡片右上角是一个小型 segmented control / pills：

评分构成 | 结构任务 | TODO

卡片正文区根据当前选中的 pill，切换内容：

评分构成：展示现在的评分拆解（结构 + 活跃 + 质量 + 任务…）

结构任务：展示你现在那一大段 checklist

TODO：从 Block 提升来的 Todo 汇总

时间线不要再出现在这里了，它已经有一个独立的「时间线」顶级 tab。这里最多放一个小链接：
查看完整时间线 → 切到上面的「时间线」tab。

这样一来：

页面只有一条真正的 tab 栏（顶部那条）。

概览内部用的是「卡片局部切换」，视觉比顶部 nav 弱得多，不会抢主角。

没有重复的“时间线”，信息层级更干净。

给你一个大致的布局示意（文字草图）：

[ MATURITY 进度条 + Stage / Score 一整块，保持现在的卡片布局 ]

------------------------------------------------------------

[ 评估 · Maturity insights                      (pills) ]

┌──────────────────────────────────────────────┐

视觉上可以这样做：

顶部 tab：保持现在这种大圆角 pill，非常「wordloom」。

内层 pills：

尺寸小一号（比如 text-sm / 高度更矮）

背景只是浅灰 hover / 选中才填充主色

靠右对齐在卡片标题行里，显得是“过滤器”，不是主导航

三、怎么跟 Copilot 说（可以直接贴给它）

你可以这样描述需求（稍微改成英文也行，这里用中式伪指令）：

保留页面顶部的主 tab：概览 / 块编辑 / 时间线，不要再新增第二条全宽 tab。

在「概览」页面里，新增一个评估卡片组件 MaturityInsightsCard：

卡片头部：左侧标题“评估”或“Maturity insights”；

右侧放一个小型 tab / segmented control，包含 3 项：
ScoreBreakdown、StructureTasks、TodoSummary。

这三个小 tab 只影响卡片内部的内容区，使用统一的容器：

ScoreBreakdown：展示现有的“查看评分构成”内容；

StructureTasks：展示原来的“结构任务”列表；

TodoSummary：展示 Block 中提升出来的 TODO 列表；

移除当前概览页面中那条全宽的 查看评分构成 / 结构任务 / 时间线 / TODO tab 栏。

内层 tab 的样式需要明显弱于顶部主 tab：

尺寸更小；

边框或浅灰底；

使用相同的主题色作为选中态，但不要投影/浮起。

在需要查看完整时间线的地方（比如结构任务或评分构成旁边），提供文案链接：
查看完整时间线，点击后切换到顶部主 tab 中的「时间线」页面。

四、要不要再进一步？

如果你以后觉得“结构任务”也太重，可以再抽象成：

评分 tab 下：所有数值型拆解

任务 tab 下：结构任务 + TODO 合并显示（上面是“推荐动作”，下面是“待办”）

但这个可以等你真的有一堆书要维护的时候，再看使用体验来调整。现在这版已经足够清楚，不会再出现「双 tab 打架」的问题。

总结一下：
顶上只保留「大模式」tab，下面所有细节都放进卡片内部的小切换里。
Wordloom 要有的是“图书馆式的安静秩序”，不是浏览器开了两排标签页那种焦虑感。
