你这个「New Library」侧栏现在怪的地方，大概有几条：

是个侧栏，但长得像没对齐的半个弹窗；

字段很少，却占了一整个高高的灰板；

主按钮在左边、靠下，和页面其它按钮节奏不一致；

背景被完全压暗，整块有点“压着”的感觉。

图本身不丑，只是不太“成熟产品”的感觉。我们可以在不大改交互的前提下，把它收拾成一个很符合 Wordloom 气质、又常见的模式。

方案一：做成标准「居中弹窗」（最稳妥、最好实现）

这个是 Notion / Google Drive / Figma 都用的套路，简单粗暴、成熟度高：

交互：

点 + New → 居中弹出一个对话框。

宽度大概 420–480px，高度按内容自动。

背景微微变暗，但还能看见底下的 Library 卡片（保持一点上下文）。

布局调整建议：

标题：

Create library / 新建书库 放在弹窗顶部，下面一行放一句小说明：

为你的知识空间取个名字。以后可以随时修改。

字段：

Name

Description（可选，文本域高度 2–3 行即可）

按钮区（非常关键）：

右下角：Cancel（左边、弱化为文字按钮） + Create（右侧、主色按钮）

这跟你页面右上角的 + New 一条视觉线，更自然。

关闭方式：

右上角一个小“×”，

点击遮罩或按 Esc 也能关。

为什么适合 Wordloom：

Library 作为“正门入口”，新建动作频繁，但表单很短，用居中弹窗不会显得厚重；

初次使用时，这个弹窗就像“欢迎来到新书库”的微型仪式感——书房风产品很适配。

标题统一术语

页面右上 + New 现在是“New”，弹窗里是 “Create Library”；

建议改成：+ New library / 新建书库，更明确。

文案更“书房一点”

Name placeholder：Enter library name → 可以换成：

给书库起个名字 / Name your library

Description placeholder：

比如：翻译项目、论文阅读、工作日报… 这类引导，更贴近实际。

提交后的体验

成功后：

关闭弹窗；

新建的 Library 卡片插到顶部并轻微高亮一下 300–500ms；

失败：

弹小 toast，输入框保留用户的输入，不要清空。

总结一句帮你拍板

如果你追求“成熟产品范儿”：
把现在这个左侧大灰板改成标准居中弹窗，字段不变，只调整布局和按钮位置，几乎所有用户一眼就懂。
