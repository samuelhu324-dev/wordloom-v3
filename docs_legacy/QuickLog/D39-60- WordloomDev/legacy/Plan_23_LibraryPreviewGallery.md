咱们拆两块说：技术上怎么不糊，和 审美上怎么好看又耐看。

一、技术层面：大厅级封面不要用 thumb

思路是：同一张封面要准备“两个规格”——大图给大厅卡片，小图给列表 / 小组件。

1. 先决定 UI 上到底要多大

比如你现在卡片大概是这种感觉（大致数一数就行）：

显示宽度：≈ 420–500px

显示高度：≈ 220–280px

也就是一个近似 16:9 的横图 banner

那按照常规 Web 习惯：

显示多大，就准备至少 2 倍宽度的素材

如果卡片显示宽度 480px
→ 真实图片建议至少 960 × 540（2x Retina）

如果考虑 4K 屏/放大一点，也可以封顶在 1280 × 720 左右

这样浏览器缩小显示会变得更锐利，而不会“放大拉伸”。

2. 封面文件应该分三种尺寸（逻辑上）

你可以在存储/命名上这样规划（不一定要三列字段，概念上有就行）：

cover_original：用户上传的原图（限制最大边，比如 2000px）

cover_large：给 Library 大卡片用，比如 1200×675 左右

cover_thumb：给小头像 / 列表 / 侧边栏用，比如 320×180

现在你看到发糊，很可能是：

只生成了 cover_thumb，然后在横向卡片里也用了这张小图；

或者后端/前端在上传时直接把图压得太狠（尺寸和质量都太低）。

解决办法：

上传后用后端或前端工具生成两套尺寸：

大图：长边 1200，质量 70% 左右（JPEG/WebP）

小图：长边 320

在卡片组件里：

大厅卡片用大图（cover_large）

列表模式名称前的小头像用小图（cover_thumb）

CSS/组件注意：

固定一个宽高比（比如 16:9）

object-fit: cover;

不要被拉伸变形

这样只要原始图还行，卡片里的封面就不会糊。

二、审美层面：厅级封面怎么做才耐看

你这个页面其实就是“Wordloom 大堂的海报墙”，可以参考 Notion / Netflix / Kindle 的思路：

1. 统一画幅 + 统一气质

相同的宽高比例（16:9 或 3:2，不要乱来）
统一之后看起来很像一排海报墙，整齐舒服。

画面尽量“有一个主体，有方向感”，不要全是花里胡哨的小细节：

比如一扇窗、一本书、一束光，而不是满屏花纹。

2. 给封面加一层“系统滤镜”

即便用户上传乱七八糟的图，你也可以用一层统一的处理让它们变成“Wordloom 风格”：

轻微压暗 + 渐变蒙版（上暗下亮，或者左暗右亮），文字区域更清晰；

略微偏冷/偏暖的统一色调（比如轻微蓝灰 → 有点“图书馆 + 金属”的质感）；

可以弄个极轻微的模糊/噪点，把照片太锐的部分柔和一点。

技术上就是前端叠一层 linear-gradient + mix-blend-mode 或在生成图片时直接套 LUT / 滤镜。

3. 封面上的文字要非常克制

卡片下面已经有 Library 名称了，封面上可以：

完全不写文字，只是图；

或者只在中央隐约放一个很淡的 WORDLOOM / Library 首字母，透明度很低。

过多文字会和下面的标题打架，而且缩小之后会显得脏。

4. 默认封面：渐变也可以更“电影感”一点

你现在的默认 cover 是粉绿渐变，这个很柔和，但可以再有点“空间感”：

加一点亮暗对比（左上亮 → 右下暗），模拟光线；

中心稍微偏亮，四角压暗一点 → 看起来像“灯光打在书桌上”。

这类图可以完全纯 CSS 画出来，甚至不需要真实图片。

三、给你一个“最终策略版”方便你拍板：

技术侧

大厅卡片封面：实际存储尺寸至少 ~1000×560，由 cover_large 提供；

列表视图小头像：用 cover_thumb（~320×180）；

组件固定比例 + object-fit: cover，禁止放大小图。

设计侧

一律横向 16:9 画幅；

用统一的滤镜 / 渐变做“Wordloom 调色”，把乱七八糟的图打成一个风格；

封面文字极简，主体视觉交给图像本身。
