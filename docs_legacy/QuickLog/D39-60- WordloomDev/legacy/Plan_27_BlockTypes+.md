好，来数一数“现代 block 编辑器动物园”里都有什么生物。你说的 title / 普通段落 / 图片我就不重复了。

我按用途分一波，方便你以后挑选放进 Wordloom。

一、常见文本类 Block（超出普通段落的）

列表系

Bulleted list（无序列表）

Numbered list（有序列表）

Check list / To-do（带勾选框的列表）

Toggle / Accordion（可折叠列表项）

强调/结构

Quote（引用段落）

Callout / Info box（带图标的提示框：⚠ 注意 / 💡 小贴士）

Divider / Rule（分割线）

Table of contents（自动目录，按页面标题生成）

代码 / 技术文本

Code block（多行代码，语法高亮）

Inline code（行内小代码片段）

Math / Formula（LaTeX 数学公式）

二、布局与容器类 Block

这些是“装别的 block”的 block：

Columns（列布局）

把页面分成 2–3 列，每列里面再放普通 block。

Notion / Craft 常见。

Group / Section（分组）

一块区域有边框 / 背景，用来包几段内容。

比如“本周重点”“附录”。

Toggle / Expandable section

一整段内容折叠收起，点一下展开。

很适合「详细解释 / 例句 / 附加说明」。

Tabs（少数工具有）

一块区域里面多个 tab，每个 tab 一堆 block。

相当少见，但一些知识库工具会玩。

三、媒体/嵌入类 Block（除纯图片以外）

文件附件 / 下载

上传 PDF、PPT、Excel 等，显示为一个卡片或行，点击打开。

音频 / 视频

内嵌播放器（播讲座录音、课程视频等）。

Embed / Bookmark 卡片

粘贴网页链接 → 自动生成预览卡：标题 + 描述 + 小缩略图。

Notion / Slack 都有类似的“链接卡片”。

画布 / Whiteboard

自由画线条、流程图、贴便签的那种（Miro 式）。

四、结构化数据类 Block

这类和你的 Wordloom 以后很容易撞上：

表格（简单版）

像 Excel，但通常只支持基础文本/数字/日期。

适合做小型记录：词表、术语表。

数据库视图 / Data view

这一类是 Notion 的杀手锏：

Table view / Board (Kanban) / Gallery / List / Calendar / Timeline

本质是：一张底层表 + 多种展示方式。

Kanban / Board

列代表状态，卡片可拖拽（Todo → Doing → Done）。

Calendar / Timeline

以日期/时间轴为主轴展示条目。

五、知识管理与引用类 Block

跟 Wordloom 的“翻译 / 术语 / 文献”天然很贴：

Mention / Reference

@某个 Page / @某人 / @某个日期

在你这可以变成：@某个 Library / Book / 术语条目。

Tag / Chip

一个小小的彩色标签，比如「法律」「技术」「金融」。

本身可作为 block，更多时候是 block 的属性/内联元素。

Quote-of-page / Embed-page

内嵌另一个页面的简略版本（只展示标题/摘要）。

Comment / Annotation（更像元数据）

对某个 block 加批注、对齐建议、翻译备注等。

六、交互式 / 动作类 Block

这类相对小众，但你将来可能会想用在自动化上：

Button / Action block

点击执行某个动作：触发脚本、生成模板、插入某种结构。

Notion 已经开始有「按钮生成内容」的雏形。

Form / Input

在页面里放一个小表单，收集数据（标题 + 选择 + 提交）。

更偏向“应用式文档”。

七、结合 Wordloom：现在值得优先收的“基础 block 集合”

根据你现在的目标（翻译 + 笔记 + 结构化知识），我会建议：

第一批必须有：

Paragraph（你已经有）

Heading 1–3

Bulleted / Numbered list

Todo list

Quote

Code block（翻译技术文章太常见）

Divider

Callout（用来写「翻译注意事项 / 术语说明」）

第二批（等主线稳定后再做）：

Toggle / Accordion（收起例句、细节解释）

简单 Table

Tag / Chip（主题、语域、来源）

Mention / Reference（指向 Library / Book / 术语表）

File / Bookmark 卡片（链接外部文章）

更后面：

Data view / Kanban / Calendar 这种就交给你“Bookshelves / Workbox / Orbit 的数据库视图”去承载，不一定放到 Block 层来。

一句压缩版总结：

现代编辑器的 block 类型，大致分成：
文本结构（列表、引用、callout、代码）、布局容器（列、折叠）、
媒体/嵌入（文件/音视频/链接卡片）、
结构化数据（表格、看板、日历）、
引用与交互（mention、tag、按钮）。
对 Wordloom 来说，先把文本结构 + 少量引用/表格打好，再考虑数据库视图和复杂交互，会更稳更符合你的主线。
