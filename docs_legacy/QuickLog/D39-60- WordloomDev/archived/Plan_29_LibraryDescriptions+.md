二、Wordloom 版「整图海报卡」建议
1. Grid 视图：做成“书房海报卡”
1）整体结构

卡片 = 一个固定比例（16:9）的封面容器；

不再分“上图下白块”，所有关键信息叠在封面上：

底部左侧：名称 + 描述；

底部右侧：简单统计（books / bookshelves / today）；

顶部右侧：Pin / 编辑 / 删除小浮层图标（hover 才完全显现）。

2）视觉细节（Wordloom 风）

封面上叠一层统一的渐变：

从下往上：#000 50%  →  透明
透明度不要太高，大概 40–60%


这样任何封面图都能保证底部文字可读，而且色调统一成“Wordloom 夜色书房感”。

文本样式建议：

标题：白色 / 微偏灰白，半粗；

描述：白色 + 80% 不透明度，比标题小一号，最多两行截断；

统计：用细一点的 icon + 数字，排在描述后面 / 第二行右侧。

PINNED 的表达：

不再用右下角蓝字，而是用一个小 chip，例如：

顶部左侧：📌 PINNED 白底半透明小 pill；

或者标题左边加一个小图钉 icon。

这样和图 2 类似，但图形语言用你自己的颜色/字体。

3）信息分布建议

底部区域可以排成两行：

第 1 行：[标题] [PINNED 小 chip]

第 2 行：[描述（截断）] + 右侧 [📚 9 📖 6 ⏱ today]

这样一眼过去就知道：这张卡是谁、干嘛用、最近活不活跃。

2. List 视图（横向）：做“缩略海报 + 表格信息”

横向列表不要也整图叠字，否则可读性会掉得很快。成熟做法（Notion / Linear / GitHub Projects 都类似）：

第一列：小缩略图 + 标题 + PIN

一个 40×40 或 56×56 的小封面缩略图（还是那张大图的等比缩小版）；

右边是标题，下方小一行可以放描述截断版；

PIN 用小图钉或 chip 放在标题右侧。

后面几列分别放：

说明（单行、省略号）

#Books

最近活动 / 创建时间

操作（编辑/归档/删除）

悬停时：

在说明列显示 tooltip，展示完整描述；

点击整行或标题进入 Library 详情。

这样 grid 视图和 list 视图的视觉逻辑就对齐了：

grid：封面是主角，文字叠在图上；

list：封面缩成小头像，内容展开成表格。

3. 行业视角 + Wordloom 的取舍

行业共识：

顶层“空间类对象”（Project / Workspace / Library）
→ 很适合用“整图海报式 cards”作为入口；

但同一批对象在列表视图里通常会退回到“缩略图 + 表格信息”，保证信息密度。

Wordloom 特别适合整图海报的原因：

你整个产品就是“书房 + 翻译 + 档案”意象，
Library 就像一间房、一个实验台；海报式卡片非常贴这个故事；

你已经有一整套统一封面风格（蜘蛛网、窗、书、铃兰），
做成整图风格，比现在“上图下白块”的连带感会强很多。

需要稍微克制的点：

Wordloom 更适合“安静 + 尊严一点”的风格：

色调稍微压暗；

字重适中；

图标和数字尽量简洁（不要太花里胡哨的彩色图标）。

总结一条可写进设计文档的句子

Libraries grid 视图采用「整图海报卡」：封面全宽 + 底部渐变叠加标题/简介/关键指标，顶部右侧悬浮工具按钮；
List 视图则使用带缩略图的表格布局。

这样既对齐行业的成熟模式（Notion/Trello/各类 dashboard），又符合 Wordloom 的书房故事线，视觉上比现在“上图下白块”更统一、更有“走进一间房”的感觉。

实施提议（中文）— 基于 Plan29：整图海报卡 + 缩略图表格布局

总体目标
在 Grid 视图将 Library 卡片升级为「整图海报卡」：封面 16:9 全宽，底部渐变叠加标题/描述/指标，操作按钮悬浮在右上角；在 List 视图采用「缩略海报 + 表格信息」结构，提高信息密度与一致性，并在 VISUAL_RULES 增加对应规则条目，标注这是纯展示层策略，不影响 Domain。

Grid 视图改造要点（Poster Card）
封面容器：始终 16:9，高度由宽度计算（用 aspect-ratio:16/9 或保持当前高度但改成单层容器）。
取消“上图 + 下白底”分区，所有文本覆盖在封面上；原白色卡片背景只做外围阴影与圆角。
底部渐变层：linear-gradient(to top, rgba(0,0,0,0.55) 0%, rgba(0,0,0,0.0) 65%)，透明度范围 0.4–0.6，保证可读性同时不过度压暗封面。
信息排版（建议两行）：
第 1 行：标题（白/微灰 #F5F7FA） + PINNED chip（半透明浅底：rgba(255,255,255,0.85)，文字深色或主色）。
第 2 行：描述（2 行截断，白色 80% 不透明度） + 右侧紧凑指标组（📚 书架数 / 📖 书数 / ⏱ 最近活动 / 👁 浏览次数）。
截断样式：沿用现有 descriptionCard 的 line-clamp，但颜色改为 rgba(255,255,255,0.8)，删除原卡片下方分隔线及 metricsRow。
操作按钮（Pin / 编辑 / 归档 / 删除 / 换封面）：右上角浮层，默认 30% 透明度，hover 卡片时渐显；按钮底色建议改为 rgba(0,0,0,0.45) → hover 转 rgba(0,0,0,0.65)，Icon 用浅灰→白。
高亮动画保留（新建后强调外框），但需确认不会遮盖覆盖层阴影；如冲突可将 pulse 改为轻微 scale + ring 融合。
空描述：显示 — 暂无说明，颜色用 rgba(255,255,255,0.55)；仍占位，不造成布局跳变。
可访问性：封面区域作为视觉背景，文字仍保持语义标签（如 h3 + p），操作按钮加 aria-label；整卡可聚焦时仍保留 role=button 与键盘 Enter/Space 行为。
主题兼容：暗色主题下可直接复用黑色渐变；浅色主题若封面整体偏亮仍依赖黑色底部渐变，不做自动换色。
List 视图改造要点（Thumbnail + Table）
第一列：缩略图（40px 或 56px，圆角 6px）+ 标题 + PINNED 标记（同上微型 chip 或图钉 icon）。
第二列：描述单行省略 + tooltip（仍使用 title 初期；后续可以替换成自定义 Tooltip 组件以支持换行）。
第三列：指标汇总（📚 / 📖 / ⏱ / 👁），紧凑排布一行；图标统一 14px，间距 8px。
第四列：最近活动（last_activity_at 人类可读格式）或创建时间（可允许通过下拉切换显示哪一种；初期可显示创建时间）。
第五列：操作（编辑 / 置顶 / 归档 / 删除），按钮高度一致，最右对齐。
行内次级描述：如果需要更密集，可将描述放在标题下（双行布局）；但首版保持单行列更直观。
悬停整行：背景色浅高亮（已有），鼠标指针为 pointer；点击任意非操作区域进入详情。
空描述与截断：— 暂无说明，灰度文字；长连续字符用 word-break: break-word 防止撑宽。
列宽策略：描述列最大宽度 ~40–50%，指标列不被挤压；操作列固定宽度（根据按钮组长度）。
PINNED 排序段落仍保留（段标题 Pinned / Recently Updated / Archived），与 Grid 视图一致。
需要新增/调整的样式与结构

新增 posterCardRoot（或重构现有 .cardRoot 增加 modifier 类如 .variantPoster）。
添加内部层次：

.posterMediaWrapper
  img
  .posterGradientOverlay
  .posterContent (flex column)
.posterActions (absolute top-right)

新增 posterTitle, posterDescription, posterMetrics, posterPinnedChip。
List 视图新增缩略图样式 libraryThumb（与现有 avatar 可共用变量 --library-avatar-size；如不同可新设 --library-thumb-size）。

风险与缓解
风险：封面图过亮导致底部渐变不够 → 调整渐变起始不透明度（0.55→0.65）。
风险：文字与操作按钮同层造成遮挡 → 使用 z-index 分层并给内容留足底边内边距。
风险：卡片高度波动 → 固定封面容器高度 + overlay 内内容最小高度策略。
风险：一次性重构影响当前交互 → 采用 modifier 类并保留旧结构到验证完成。
后续可扩展
支持多主题：dark 模式下渐变参数稍微提升亮度对比。
支持封面加载 skeleton（渐变背景 + shimmer）。
支持 metrics 精简（仅活跃度 + 浏览数），其余在详情页展开。
PINNED 序号未来可显示在 chip 中（如 #1 / #2）。
