一、通用原则（给两个视图都用）

说明只是 tagline，不是正文

设计上就当“一句副标题”来用，而不是给人写小论文的。

强制“最多 N 行 + 省略号”

卡片可以给 2–3 行机会；

列表只给 1 行（信息密度更重要）。

全文要有地方看，但不要挤在列表里

完整说明放在：

Library 详情页；或

鼠标悬停的 tooltip / 气泡；或

编辑弹窗 drawer 里。

下面具体拆开说。

二、卡片视图（grid / card）怎么救
1. 说明最多 2 行 + 渐隐 + 省略号

成熟做法可以参考 Notion / Trello 卡片：

样式上类似这样：

字体：比标题小一号（例如正文 14，描述 13）；

颜色：次要文字色（灰一些）；

限制：最多 2 行，超出部分隐藏并加 … 或渐隐。

实现思路（伪 CSS）：

.library-card-description {
  font-size: 0.9rem;
  color: #6b7280;           /* 次要文字色 */
  display: -webkit-box;
  -webkit-line-clamp: 2;    /* 限制 2 行 */
  -webkit-box-orient: vertical;
  overflow: hidden;
}


这样即使像你截图那样疯狂“阿阿阿”，也只会撑到两行，然后乖乖被剪掉。

2. 在说明末尾给一个“小尾巴”：更多 / icon

为了让用户意识“这句话还有后半截”，可以两种选法：

轻量：什么都不加，用户 hover 卡片时，整张卡片的 tooltip 显示完整说明；

稍重一点：在说明右下角出现一个「更多…」小文字 / … 图标，点击打开 Library 详情。

Wordloom 比较“静一点”，我会偏向：

默认只截断，
hover 整张卡片或说明区域时，用 tooltip 显示完整说明。

三、横向列表视图（table）怎么救

列表的目标是信息密度，所以这里可以更狠一点：单行截断。

1. 说明列：单行 + 省略号 + 鼠标悬停查看

建议行为：

说明列永远只显示一行：

white-space: nowrap;

overflow: hidden;

text-overflow: ellipsis;

列宽可以固定一个最大宽度，比如 40–50% 宽度，再多的文字直接省略。

鼠标悬停在这一列时：

显示 tooltip：标题 + 完整说明；

或者直接在这一列的 tooltip 里只放全文说明。

视觉上就变成类似：

Hello 阿阿阿阿阿阿阿阿阿阿阿阿阿…

而不是你现在那条蛇直接伸到操作按钮底下。

2. “说明 / 指标”分拆心智

你现在列表头是「说明 / 指标」，下面又有那一行小图标（books、bookshelves、today、views），容易让人搞不清这列到底承载什么。

一个更清晰的拆法：

列：

说明：只放文案，按上面的“单行 + 省略号”处理；

统计：把那一行图标数据放在这一列里（或者放到名称下面一行）。

或者干脆换一种布局：

第一行：封面 + 名称 + PINNED；

第二行（较小字体）：说明（截断） + 那些统计 icon；

操作按钮固定在最右侧一列，不被文本挤走。

这样不管说明多长，操作按钮的位置都很稳定，列表不会“抖”。

四、两个视图的一致性小规则

为了让用户从卡片视图切到列表视图时不懵，可以定几条“说明的行为公约”：

两边截断规则一致

卡片：最多 2 行；

列表：最多 1 行；

都用省略号 + tooltip 展示全文。

说明永远是“第二优先级信息”

标题（名称）始终一眼能看到，不能被挤；

说明截断没关系，只要用户能通过 hover / 详情看到就行。

详情页是完整信息的归宿

只要点进 Library 详情页，就给一整段说明，支持多行、换行、甚至 markdown；

列表和卡片始终只是摘要。

五、如果要更“Wordloom 味”一点

可以顺手加两点小趣味：

当说明太长被截断时，在 tooltip 里加一句很淡的小字：

「这一段在说明里被折叠了，你可以在详情页里看到全部内容。」

对于完全没写说明的库：

列表 / 卡片上用一个淡灰占位：

— 暂无说明

或「这个书库还没有简介。」

这样既稳又不会显得“土”，跟你现在的 UI 风格也比较贴。

压缩版结论：

**卡片视图：**说明限制 2 行，超出用 line-clamp + 渐隐/省略号，完整内容放 tooltip / 详情页。

**横向列表视图：**说明限制 1 行，固定最大宽度，text-overflow: ellipsis，悬停看全文，不让说明挤到创建时间和操作按钮。

照这个改，你截图里那条“阿阿阿阿阿阿蛇”会被截得干干净净，两个视图都还能保持优雅。
